# SQLAlchemy Reserved Words

## Common Reserved Words to Avoid as Column Names

### SQLAlchemy Reserved
- `metadata` - Used for table metadata
- `query` - Used for query objects
- `select` - SQL SELECT keyword
- `update` - SQL UPDATE keyword
- `delete` - SQL DELETE keyword
- `insert` - SQL INSERT keyword
- `create` - SQL CREATE keyword
- `drop` - SQL DROP keyword
- `alter` - SQL ALTER keyword
- `table` - SQL TABLE keyword
- `column` - SQL COLUMN keyword
- `index` - SQL INDEX keyword
- `key` - SQL KEY keyword
- `primary` - SQL PRIMARY KEY
- `foreign` - SQL FOREIGN KEY
- `unique` - SQL UNIQUE constraint
- `check` - SQL CHECK constraint
- `default` - SQL DEFAULT keyword
- `null` - SQL NULL keyword
- `not` - SQL NOT keyword
- `and` - SQL AND operator
- `or` - SQL OR operator
- `in` - SQL IN operator
- `between` - SQL BETWEEN operator
- `like` - SQL LIKE operator
- `is` - SQL IS operator

### Python Reserved Words
- `class`
- `def`
- `import`
- `return`
- `yield`
- `global`
- `nonlocal`
- `lambda`
- `with`
- `as`
- `assert`
- `del`

## Alternative Names to Use

Instead of `metadata` → Use `meta_data`, `metadata_info`, `data_metadata`
Instead of `permissions` → Use `perms`, `user_permissions`, `access_rights`
Instead of `settings` → Use `preferences`, `user_settings`, `config`
Instead of `type` → Use `type_name`, `category`, `kind`
Instead of `status` → Use `current_status`, `state`
Instead of `date` → Use `date_created`, `created_date`
